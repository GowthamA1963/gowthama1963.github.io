What is DWA?

The Dynamic Window Approach (DWA) is a local path planning algorithm designed for mobile robots. It works by computing velocity commands in real-time based on the robot's current state and sensor data. The algorithm searches through a "dynamic window" of achievable velocities, considering the robot's kinematic constraints and obstacle avoidance requirements.

DWA evaluates multiple trajectory candidates and selects the optimal one based on a cost function that balances:

  • Progress toward the goal
  • Obstacle clearance
  • Velocity smoothness

Project Overview

This repository contains a custom implementation of the DWA local planner for ROS2, specifically designed for the TurtleBot3 robot in a Gazebo simulation environment. The planner computes velocity commands (cmd_vel) based on sensor data including LaserScan and Odometry, providing smooth navigation while avoiding obstacles.

Motivation

I built this project to deepen my understanding of path planning fundamentals in ROS2. By implementing DWA from scratch, I gained hands-on experience with:

  • Local trajectory optimization
  • Sensor data processing (LaserScan, Odometry)
  • Velocity command generation
  • Real-time obstacle avoidance
  • ROS2 node architecture and communication

Technical Implementation

The planner subscribes to sensor topics and publishes velocity commands:

  • Input: /scan (LaserScan), /odom (Odometry)
  • Output: /cmd_vel (Twist - linear and angular velocities)
  • Algorithm: Dynamic window search with trajectory scoring
  • Environment: Gazebo simulation with TurtleBot3 Waffle Pi
