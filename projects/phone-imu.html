📱 Phone IMU → ROS2 Sensor Fusion

Real-time IMU sensor fusion using a smartphone as the sensor, ROS2 as the middleware, and a complementary filter to estimate 3D orientation.

Architecture

The system utilizes the Phyphox app to stream IMU data (accelerometer and gyroscope) over WiFi via a REST API. A custom ROS2 node polls this endpoint at 50Hz, publishing raw data to the /imu/data_raw topic. 

Phone (Phyphox App)
      ↓  HTTP REST @ 50 Hz over WiFi
phone_imu_node  →  /imu/data_raw  (sensor_msgs/Imu)
                          ↓
                  imu_filter_node  (Complementary Filter)
                          ↓
                    /imu/data  (sensor_msgs/Imu + quaternion)
                    /imu/markers  (visualization_msgs/MarkerArray)
                    TF: world → phone_imu
                          ↓
                        RViz2

Sensor Fusion — Complementary Filter

The filter fuses gyroscope and accelerometer data to estimate roll, pitch, and yaw. It leverages the short-term accuracy of the gyroscope and the long-term stability of the accelerometer.

Source Short-term Long-term
Gyroscope ✅ Accurate ❌ Drifts
Accelerometer ❌ Noisy ✅ Stable tilt
Fused (α=0.98) ✅ Smooth ✅ Drift-corrected

Default alpha 0.98 = 98% gyro trust, 2% accelerometer correction per step.

Key Features

  • Real-time Streaming: Low-latency data ingestion from mobile sensors.
  • Visual Feedback: RViz2 markers that change color based on angular velocity.
  • TF2 Integration: Publishes dynamic transforms from world to phone frame.
  • Modular Design: Separated data acquisition and filter nodes for better ROS2 architecture.
